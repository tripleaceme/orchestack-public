Services / dbt Core / Edit config

Edit dbt Core config

Change the GitHub repo URL, target database, or production schema. Changes take effect on the next container start. So stop and restart dbt from the Services page after saving.

⚠
Service must be stopped to apply. dbt is currently stopped, so you can save freely. If it were running, the Stop button would appear at the bottom of this form before Save.

Repository

Where dbt clones its project from on every start
If your dbt project is on GitHub, paste the HTTPS or SSH URL. OrcheStack clones it into the dbt container on first deploy and refreshes it on every run. Leave blank to use the built-in demo project (warehouse_tables).

Warehouse connection

profiles.yml is regenerated from these values on every start
Default is your warehouse DB. Override only if you want dbt to write to a dedicated analytics DB.
