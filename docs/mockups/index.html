Dashboard redesign — 3 directions

Each direction reimagines the Services + Credentials pages while keeping the existing colour tokens (--navy, --accent, --surface, --border). Click through each, then tell me which feel wins.

OrcheStack mockups · phase 3.7-explore
Direction A

Deployment Board

"The calm scan"
Inspired by: Vercel deployment list, GitHub Actions runs
Visual feel:
  • Vertical rhythm — each service is a single row, lots of whitespace
  • Binary status — green dot or grey; nothing else competes
  • Single column at max ~920px, never feels noisy
  • Type-driven hierarchy — service name large, metadata small
"what's running right now?"

Direction B

Operations Cockpit

"The KPI-first overview"
Inspired by: Stripe overview, Mercury, Datadog (lite)
Visual feel:
  • KPI strip — 4 large numbers up top answer "is everything healthy?"
  • Sparklines for trending values (postgres connections, etc.)
  • 3-column grid for services — compact cards with state + actions
  • Grouped by state — Running first, Stopped below
"is the platform healthy?"

Direction C

Power-User Console

"Get to the thing fast"
Inspired by: Linear, Plane, Notion
Visual feel:
  • Sidebar nav — keyboard shortcut hints inline
  • Dense table rows grouped by tier (Hot / Cold)
  • Keyboard-first — ⌘K command palette + J/K navigation
  • Maximum info density — everything visible without scroll
"how do I get there in two keystrokes?"
