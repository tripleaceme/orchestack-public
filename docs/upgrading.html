Upgrading OrcheStack

How to safely move from one OrcheStack version to the next without losing data or breaking your pipelines.

Versioning

OrcheStack follows semantic versioning. Docker images are pinned by tag in docker-compose.yml — never use :latest in production.

Upgrade steps

  1. Read the release notes at github.com/tripleaceme/OrcheStack/releases for breaking changes.
  2. Take a full backup (see Backup & restore).
  3. Edit docker-compose.yml to bump image tags.
  4. Run docker compose pull to fetch the new images.
  5. Run docker compose up -d. Services with migrations will run them at startup.
  6. Verify the Streamlit dashboard loads and the nightly DAG succeeds on its next run.

Rollback

If something breaks: revert docker-compose.yml to the previous tags, restore from backup, and file an issue.
