Step 4 — Run your first pipeline

With OrcheStack configured, trigger your first end-to-end data pipeline — ingest, transform, test, serve — and verify the results land in Metabase.

Load the demo dataset

OrcheStack ships a demo dataset — a smallholder agricultural cooperative with yield records, market prices, and supply-chain events — to let you run the pipeline end-to-end without connecting a real source first.

From the Streamlit admin dashboard, click Load demo data. OrcheStack does three things:

  1. Drops the demo CSVs into MinIO under s3://OrcheStack/raw/demo/.
  2. Configures an Airbyte connector that watches that bucket for new files.
  3. Creates a pre-built dbt project under ./dbt/ with staging and marts models for the demo schema.

Trigger the pipeline

Click ▶ Run pipeline now on the admin dashboard. The Orchestrator:

  1. Spins up Airbyte (cold tier) and syncs the demo CSVs into PostgreSQL raw schema.
  2. Spins up dbt (cold tier) and runs dbt build --select tag:demo, which builds stg_yields, stg_prices, and fct_coop_performance in the marts schema.
  3. Runs Great Expectations on the mart — checks that yields are positive, prices are not null, and the state column matches the 36 Nigerian states plus FCT.
  4. Runs Elementary to record test results.
  5. Shuts down the cold services.

Monitor the run

While the pipeline runs, Streamlit shows live progress. Each stage streams logs into the dashboard. You can also watch individual task states in the Airflow UI — click Open Airflow on the admin dashboard.

A full run of the demo pipeline typically takes 45–90 seconds on a 4-core laptop. Your first run will be slower (~3 minutes) because Airbyte and dbt container images are being pulled.

Verify the output

When the run completes, click Open Metabase. The demo dashboard shows:

  • Total yield by state (bar chart)
  • Average market price by crop (trend line)
  • Cooperative performance scorecard

If you see numbers, your pipeline is live. You've proven the end-to-end path: raw CSV → MinIO → PostgreSQL raw → dbt marts → Metabase dashboard.

Schedule nightly runs

By default, OrcheStack creates an Airflow DAG called OrcheStack_nightly that runs at 02:00 local time. You can edit its schedule from Streamlit's Pipelines page or directly in the Airflow UI. The DAG re-runs the full pipeline and emails a summary if tests fail.

Using your own data? Replace the demo connector with an Airbyte connector for your source (Postgres, Google Sheets, Stripe, etc.), update the dbt models under ./dbt/models/, and run the same pipeline. Everything else stays the same.
