Step 1 — Install OrcheStack

OrcheStack ships as Docker images on Docker Hub plus a compose file on GitHub. Most people install via a single installer script; power users clone the repo instead.

Prerequisites

  • Docker 24+ and Docker Compose v2.20+
  • Host machine with at least 4 GB RAM and 20 GB disk free (the base install uses ~2 GB)
  • Linux, macOS, or Windows with WSL2. Any POSIX-style host works.
  • Ports 80 and 443 available on the host (OrcheStack's reverse proxy binds here)

Working behind a firewall? You'll need outbound HTTPS to hub.docker.com (Docker images) and orchestack.africa (installer + compose file). Everything else is internal.

Where OrcheStack lives

OrcheStack is three artifacts in three places — understanding the split makes the install paths below obvious:

  • Docker Hub (hub.docker.com/r/tripleaceme/orchestack-*) — our prebuilt images: tripleaceme/orchestack-auth (signup, login and setup wizard), tripleaceme/orchestack-orchestrator (hot/cold service lifecycle daemon), tripleaceme/orchestack-streamlit (administrator dashboard), and tripleaceme/orchestack-airflow (Apache Airflow with dbt + Cosmos preinstalled). These are what docker compose pulls.
  • GitHub (github.com/tripleaceme/orchestack) — source of truth for the compose file, the service generator, docs, and the setup skeleton. This is what you clone if you want to read, fork, or contribute.
  • orchestack.africa — marketing + docs + the installer script you're about to run. The installer is a thin shell script that pulls from the other two places.

Pick an install option below based on how much you want to see under the hood.

Option A — Installer script (recommended)

Single command. Good for production hosts, demo laptops, and CI runners that just need OrcheStack running. The script downloads the latest release tarball, prompts you for the OrcheStack DB password, writes .env, and runs docker compose up -d for you.

curl -sSL https://orchestack.africa/install.sh | bash

What happens, step by step (you can read the script first with curl ... -o install.sh; less install.sh):

  1. Checks that docker and docker compose are installed and the daemon is running.
  2. Downloads the latest tarball from GitHub releases + verifies its SHA256 checksum.
  3. Extracts to ./orchestack/ (override with ORCHESTACK_DIR).
  4. Prompts you twice for ORCHESTACK_DB_PASSWORD (or skips this if you pre-set the env var).
  5. Writes .env from the template, chmod 600.
  6. Runs docker compose up -d — Compose pulls all 5 images from Docker Hub in parallel (~30s).
  7. Waits for the auth container to respond, prints the URL.

Useful environment overrides:

  • ORCHESTACK_VERSION=v0.1.0 — pin a specific release instead of latest.
  • ORCHESTACK_DIR=/opt/orchestack — install somewhere other than the current directory.
  • ORCHESTACK_DB_PASSWORD=... — non-interactive install (CI / automation).
  • ORCHESTACK_NO_START=1 — download + write .env but don't start containers yet.

Option B — Manual install from the release tarball

For operators who want to inspect every file before docker compose up, or who are deploying to an airgapped host. The installer script does this for you; this option is the same thing by hand.

# 1. Download the latest runtime bundle (or pick a specific version from
#    https://github.com/tripleaceme/orchestack/releases)
$ curl -fsSL -o orchestack.tar.gz \
    https://github.com/tripleaceme/orchestack/releases/latest/download/orchestack-runtime.tar.gz

# 2. Verify the checksum (optional but recommended)
$ curl -fsSL -o orchestack.tar.gz.sha256 \
    https://github.com/tripleaceme/orchestack/releases/latest/download/orchestack-runtime.tar.gz.sha256
$ sha256sum -c orchestack.tar.gz.sha256

# 3. Extract — you'll get an orchestack-runtime-${VERSION}/ folder
$ tar xzf orchestack.tar.gz
$ cd orchestack-runtime-*

# 4. Set the bootstrap password (the only thing you need to type)
$ cp .env.example .env
$ $EDITOR .env                  # set ORCHESTACK_DB_PASSWORD

# 5. Start the control plane — Compose pulls all 5 images in parallel
$ docker compose up -d

You end up at the same place as Option A. The tarball is ~30 KB — only the files you need to actually run OrcheStack. The Dockerfile, HTML source, docs, and so on stay on GitHub and don't clutter your install.

Note: You do not need to run docker pull manually. docker compose up pulls all 5 service images from Docker Hub automatically (in parallel) the first time it runs. Subsequent boots are local cache hits.

Building from source (contributors only)

If you want to modify the auth image, rebuild containers locally, or contribute to OrcheStack, clone the full repository instead. This is not an install path — it's a development path.

git clone https://github.com/tripleaceme/orchestack
cd orchestack/system/docker
cp .env.example .env            # set ORCHESTACK_DB_PASSWORD
docker compose up -d

You get the full source tree (Dockerfiles, HTML, docs, generators), which is overkill if you just want OrcheStack running. Use Option A or B for that.

Verify the install

All three paths end at the same state. Run docker compose ps in your install directory; you should see:

[+] Running 5/5
 ✔ Container orchestack-proxy         Started   (Traefik reverse proxy)
 ✔ Container orchestack-postgres      Started   (database, internal DB initialised)
 ✔ Container orchestack-auth          Started   (signup + login + setup wizard)
 ✔ Container orchestack-streamlit     Started   (M1 placeholder for the admin dashboard)
 ✔ Container orchestack-orchestrator  Started   (M1 stub for the lifecycle daemon)

Only these 5 containers run at this stage. No Airbyte, dbt, Metabase, or any other service is pulled yet — those come after you configure the platform via the browser wizard.

Open http://localhost. You should land on the OrcheStack signup page (no users exist yet).

Port conflict? If something else is using port 80 on the host, edit the PROXY_HTTP_PORT variable in your .env file and re-run docker compose up -d.
