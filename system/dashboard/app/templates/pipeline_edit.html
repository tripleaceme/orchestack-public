About the buffer: after each step's start or stop command completes, the pipeline waits this many seconds BEFORE running the next step. Gives the service time to finish booting (Airflow scheduler tick, Airbyte worker register, etc.) so the next step can rely on it being responsive. Default 300 s = 5 min. Last step's buffer is ignored (nothing follows it).
Why only cold-tier services in the dropdown: hot-tier services (Postgres, Metabase, pgAdmin) are always running, starting them via a pipeline is a no-op. Pipelines are for waking cold services before their internal schedulers fire.
