Roles & permissions

Each role grants a set of platform permissions per service. Built-in roles (Admin / Engineer / Analyst) cover the typical OrcheStack team shape; create custom roles for tighter scopes. Assign roles to users from the Users page.

Permissions are platform-level: can_start (start a service), can_use (open a session), can_force_stop (stop a service with active sessions), can_edit_config (edit credentials). Tool-internal permissions (e.g. who can edit a Metabase dashboard) are pushed to each tool when the user opens it, handled by per-tool integration logic.
