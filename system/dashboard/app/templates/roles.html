Roles & permissions

Each role grants a set of platform permissions per service. Pick a service from the dropdown, tick the permissions you want for that role on that service, save. Service permission rows can be added, updated (re-grant overrides), or revoked. The three built-in roles (Admin / Engineer / Analyst) cover the typical OrcheStack team shape; create custom roles for tighter scopes.

Permissions are platform-level: can_start (start a service), can_use (open a session), can_force_stop (stop a service with active sessions), can_edit_config (edit credentials).
Tool-internal permissions (e.g. who can edit a Metabase dashboard) are pushed to each tool when the user opens it, handled by per-tool integration logic in a later milestone.
